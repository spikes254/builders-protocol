## What This Is

A single, fully self-contained HTML file. No frameworks. No CDN calls. No accounts. No expiry.

Open it in a browser — it works. Send the file to someone — it works. Host it anywhere — it works. Disconnect from the internet — it still works.

**Built for:** self-taught developers, security researchers, and independent builders operating without institutional support — especially in emerging markets where there is no safety net.

## The Five Axioms

<details>
<summary><strong>01 · Systems over symptoms</strong></summary>
<br>
Every bug, every failed prediction, every stalled project is a signal emitted by an underlying structure. Always trace one level deeper than the symptom.
</details>

<details>
<summary><strong>02 · Build first, validate honestly</strong></summary>
<br>
A working model — even a broken one — reveals constraints no plan can predict. Build the minimum that can be tested. Then test it against real conditions.
</details>

<details>
<summary><strong>03 · Every system has an attack surface</strong></summary>
<br>
Security thinking is not a specialty. It is foundational literacy. Every system has points of failure. The builder who does not model their own attack surface has outsourced that analysis to whoever comes next.
</details>

<details>
<summary><strong>04 · Signal density is respect</strong></summary>
<br>
Every word that adds no new information is an extraction from the reader's attention. Compress everything. Write dense. Think clearer.
</details>

<details>
<summary><strong>05 · Income must compound, not consume</strong></summary>
<br>
Time-for-money is a linear trap. The goal is not to earn more money — it is to build structures that generate income while you are building other structures.
</details>

## Design Decisions

| Decision | Reason |
|---|---|
| Single HTML file | Works offline, no hosting dependency, trivially shareable |
| No external fonts | Loads instantly, works without internet, no GDPR concern |
| No JavaScript frameworks | No breaking changes from upstream library updates |
| No cookies or tracking | Nothing to expire, nothing to consent to |
| CSS variables only | Theming without preprocessors, readable without tooling |
